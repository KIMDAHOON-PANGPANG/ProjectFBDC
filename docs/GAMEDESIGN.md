# ProjectFBDC — Game Design Document

**버전**: v0.1 (2026-05-21 초안)
**개정 주기**: 마일스톤 종료 시마다 본 문서 갱신 + 다음 마일스톤 체크리스트 확정

> 본 문서는 **사용자(기획/디렉터)** 가 게임의 방향성과 진행도를 단일 화면에서 확인하기 위한 단일 출처다. 시스템 구현 세부(데이터 흐름, 책임 분리, 파일 경로)는 [ARCHITECTURE.md](ARCHITECTURE.md)로 분리되어 있다.

## 0. 비전

> **"한 손으로 30분, 한 챕터로 영원히 살아남는 사무라이."**

- **장르 핵심**: HD-2D 핵앤슬래시 × Vampire Survivors 류 메타 진행 × 인왕(Nioh) 풍 텔레그래프 전투
- **차별점**:
  - VS류의 자동공격이 아니라 **이아이도(거합)** — 차징 + 방향 조준 + 대시-슬래시. 한 번의 결단으로 다수를 베는 손맛
  - 보스/엘리트는 인왕/소울라이크처럼 **텔레그래프 + 패리/회피** 결정. 잡몹은 VS처럼 그냥 베고 다님
  - HD-2D 비주얼 (Octopath 스타일) — 3D 월드 + 2D 스프라이트 빌보드의 깊이감
- **세션 길이 목표**: 1챕터 = 2~5분 (현재 Ch1 = 120s 고정). 메타 진행으로 영구 성장
- **플레이 루프**:
  1. 아웃게임에서 챕터 선택 + 시작 패시브 강화
  2. 인게임 웨이브 생존 → 클리어 또는 사망
  3. 아웃게임으로 복귀 → 혼(재화) 정산 → 패시브/카드 풀 언락 → 다시 1번

## 1. 게임 흐름

### 1.1 아웃게임 (현재 미구현 — Milestone 1~4에서 단계 도입)

```
[메인 메뉴]
   │
   ├─ Chapter Select  ──► 챕터별 최고기록 (시간/처치/레벨) 표시 → 시작
   │
   ├─ Meta Passives   ──► 혼으로 패시브 단계 구매 (HP+, EXP+, 시작 카드 슬롯+, …)
   │
   ├─ Card Unlock     ──► 혼으로 신규 카드 풀에 추가
   │
   └─ Settings / Quit
```

### 1.2 인게임 (현재 Chapter 1만 살아있음)

```
t=0     : 일반 잡몹 드립 스폰 시작 (target 18)
t=30    : 인구 목표 39로 증가
t=60    : 잡몹 LV2 전환 + 엘리트 3종 동시 등장
t=90    : 인구 목표 55 (최대 압박)
t=120   : 인구 목표 9로 급감 + 보스 등장
보스 처치 → 챕터 클리어 → 아웃게임 복귀 (현재는 같은 챕터 재시작)
PC 사망 → 아웃게임 복귀 (현재는 R로 수동 재시작)
```

## 3. 시스템별 상세

### 3.1 플레이어 — 거합 사무라이

**기본 능력치** (`resources/player/player_data.tres`)
| 항목 | 값 | 비고 |
|---|---|---|
| 이동 속도 | 5.0 | Quickstep 카드 +20% 스택 |
| 최대 HP | 5 | Iron Will 카드 +1 스택 |
| 슬래시 사거리 | 3 ~ 11 | Reach 카드 +20% 스택 |
| 슬래시 폭 | 1.4 | Razor's Edge +30% 스택 |
| 슬래시 쿨 | 0.15s | Vigilance -30% 스택 |
| 최대 차징 | 0.72s | 튜닝 완료 (사용자 컨펌) |
| 대시(슬래시) 시간 | 0.12s | 대시 중 무적 |
| 회피 거리/쿨 | 3.5 / 0.7s | Wind Walker -25% 쿨 스택 |
| 피격 i-frame | 1.0s | 강제, 메타 강화 후보 |

**확장 후보** (Milestone 6+)
- [ ] 무기 다양화 보류 (사용자 결정 — 거합만 유지)
- [ ] 메타 패시브로 기본 능력치 영구 +X (#3.5 참조)

### 3.2 적 / 보스

| 카테고리 | HP | 행동 | EXP |
|---|---|---|---|
| 잡몹 근거리 LV1 | 1 | 추적 → 부채꼴 텔레그래프 0.5s → 1뎀 | 1 |
| 잡몹 근거리 LV2 | 2 | 동일 (t=60s 이후 전환) | 2 |
| 잡몹 원거리 | 1 | 6유닛 유지 + AimLaser 1s → 화살 | 1 |
| 엘리트 1 (빨강) | 1 | 추적 + 부채꼴, 사망 시 폭발 | 3 |
| 엘리트 2 (초록) | 3 | 동일, 사망 시 보너스 CircularSlash | 5 |
| 엘리트 3 (파랑) | 5 | 동일, 사망 시 3초 불릿타임 | 10 |
| 보스 (Ch1) | 15 | YELLOW(70%)/RED(30%) 시그널 + 부채꼴 2뎀 | — |

**확장 방침** (사용자 결정 — Milestone 6에서):
- 추가 챕터마다 보스 1종 신규 (Boss.gd 패턴 복제 + 패리 컬러/패턴 변경)
- 엘리트 3종 유지 (효과만 챕터별로 교체 가능성)
- 잡몹은 LV2 / LV3로 점진 강화 (HP / 속도 / 데미지)

### 3.4 카드 / 업그레이드 풀

**현재 풀 7종** (전부 단순 수치)
- Razor's Edge — 슬래시 폭 +30%
- Quickstep — 이속 +20%
- Iron Will — 최대 HP +1
- Wind Walker — 회피 쿨 -25%
- Greed — EXP +25%
- Reach — 슬래시 사거리 +20%
- Vigilance — 슬래시 쿨 -30%

**확장 풀 후보** (Milestone 3에서 디자인 → 5에서 언락 시스템과 통합)

*A. 수치 카드 추가 (간단)*
- [ ] Sharpened Blade — 엘리트/보스 대상 데미지 +1
- [ ] Berserker — HP가 절반 이하일 때 슬래시 폭 +50%
- [ ] Vampire — 적 처치 시 1/N 확률로 HP +1
- [ ] Whirlwind — 슬래시 쿨 추가 -20% (Vigilance와 스택)
- [ ] Featherweight — 회피 거리 +50%

*B. 메커니즘 카드 (큰 작업)*
- [ ] Multistrike — 한 번 충전으로 슬래시 2회 연속 (방향 동일, 약간 시차)
- [ ] Echo — 슬래시 후 0.3초에 같은 자리에 CircularSlash (작게)
- [ ] Reflect — 적 화살이 PC 회피 i-frame 중에 닿으면 반사
- [ ] Bullet Time on Kill — 엘리트 처치 시 0.5초 추가 슬로우
- [ ] Magnet — EXP 자동 흡수 (현재는 어차피 즉시 적립이지만, 거리 표시 / 시각 효과)
- [ ] Phoenix — HP 0 도달 시 1회 부활 (1라운드 1회)

*C. 무기/패시브 트리 (먼 미래)*
- [ ] 추가 무기는 현재 보류 (거합 단일 유지)

### 3.5 메타 진행 (영구강화) — VS 스타일 단순화

**재화**: "혼(魂)" — 챕터 클리어 시 +N, 사망 시 그 시점까지 누적 EXP×0.5 정도 적립

**메타 패시브 카테고리 후보** (Milestone 4)
| 카테고리 | 효과 | 단계 |
|---|---|---|
| 강건한 육신 | 시작 HP +1 | 1~5 |
| 단련된 검 | 슬래시 폭 +5% | 1~10 |
| 빠른 발 | 이속 +3% | 1~10 |
| 강한 영혼 | EXP 획득 +5% | 1~10 |
| 능숙한 회피 | 회피 쿨 -3% | 1~5 |
| 깨어난 자 | 시작 시 무료 카드 +1장 (시작 즉시 레벨1 카드 1장) | 1~3 |
| 인내 | 피격 i-frame +0.1s | 1~5 |

**카드 풀 언락** (Milestone 5)
- 초기 풀: 4~5장만 (Razor's Edge, Quickstep, Iron Will, Reach 정도)
- 혼 비용으로 나머지 카드 풀에 영구 추가
- 신규 메커니즘 카드(#3.4 B)는 챕터 클리어 같은 조건 + 혼 비용 둘 다

**최고기록 표시** (Milestone 1)
- 챕터별 항목: 클리어 시간(빠를수록 좋음), 최대 도달 레벨, 총 처치 수, 최고 동시 잡몹 수

## 7. 운영 / 작업 원칙

- **마일스톤 = 1~2주 단위**로 잡되, Milestone 1처럼 작은 건 며칠 안에 완료 가능.
- **마일스톤 종료 시 본 문서 갱신** — 체크박스 채우기 + 다음 마일스톤 추가 작업 발견 시 추가.
- **마일스톤 종료마다 `refactor-pass` 스킬 한 번 적용** 권장 — 기능 추가 누적이 구조를 갉아먹기 전에 정리.
- **모든 변경은 main 브랜치 직접 커밋** (메모리 `feedback_push_workflow`).
- **세계관 / 비주얼 / 사운드는 게임플레이가 안정된 뒤** — Milestone 6 종료 후가 자연스러운 시점.
