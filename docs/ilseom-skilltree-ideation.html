전투 메커니즘 스킬트리

신도 내러티브도 없다. 일섬 · 열기 · 회피 · 패리 · 젠 — 게임을 굴리는 메커니즘 그 자체를 디아블로 / PoE / VS 처럼 찍는 스킬트리로 만든다. 중앙 코어(거합 사무라이)에서 7개 브랜치가 방사하고, 각 브랜치는 minor → notable → keystone 노드 체인으로 이어진다.

1

개요 — 무엇을, 어떻게 찍나

軸메커니즘이 축

5대 전투 메커니즘(일섬·열기·회피·패리·젠)과 보조 2축(스윙 속도·생존)을 트리의 축으로 삼는다. 테마가 아니라 손에 쥔 동사 그 자체.

＋포인트로 찍기

레벨업 / 메타 진행으로 얻은 포인트를 노드에 투자. 선행 노드를 거쳐야 상위가 열린다. 예산 안에서 무엇을 포기할지가 빌드.

鍵키스톤 = 정체성

각 브랜치 끝의 keystone은 강력한 이득과 명확한 대가를 함께 준다. 한 빌드는 보통 1~2개의 키스톤이 정의한다.

混브랜치 혼합

한 브랜치에 올인하지 않아도 된다. 두세 브랜치의 minor/notable을 섞어 하이브리드 — 수류+젠, 회피+흡혈 같은 시너지 루프가 핵심 재미.

구현 우선순위

  1. 1. 데이터-드리븐 노드 정의NEW — 노드 = SkillNode Resource(id/branch/tier/prereq[]/effect_key/value). 트리 전체를 .tres 배열로. 효과는 기존 Player 런타임 보너스 변수 패턴 재사용(런마다 리셋).
  2. 2. minor 9종 = 기존 카드 재배선 — UpgradeSystem 효과를 노드 effect_key로 옮긴다. 가장 저위험, 즉시 가치.
  3. 3. notable 능동/조건 효과 — 레드존·관통·반격의 호흡·受け熱 등. Player에 조건 훅(열≥80%, 퍼펙트닷지 직후 등) 추가.
  4. 4. keystone 7종 + 대가 처리 — 강한 이득 + 패널티(평타 봉인·i-frame 제거·무재생). 빌드 정체성. 밸런스 비용이 가장 큼 → 후순위.
  5. 5. 트리 UI — 본 HTML의 SVG 배치/할당 로직을 Godot Control로 이식. 메타 화면(OutGame)에 영구 트리, 인게임은 레벨업 시 인접 노드만 제안하는 축약형도 가능.

VS 하이브리드 안: 인게임은 지금처럼 카드 3택을 유지하되, 뽑히는 카드 풀을 "메타에서 해금한 노드"로 한정 → 트리 = 영구 진행, 카드 = 런 내 픽. 두 시스템을 충돌 없이 합칠 수 있다.
